Idea-to-Demo Blueprints

想法到 Demo 方案库

把想法变成真实 Demo，并把 Demo 的制作过程变成别人可以复现的路线图

一个可复现 Demo 蓝图库：每个 Blueprint 都包含原始想法、方案设计、Demo Pack、可运行 Demo、关键 Prompt、构建过程和验收记录。

📋 开源方案库 🚀 快速验证 📝 可执行蓝图 🤖 AI Workflow ✅ 3 个 demo-ready

不是灵感收藏夹，而是可复现 Demo 蓝图库

💡 想法停留在脑子里

每个人都有"如果这样做会怎么样"的想法，但大多数停留在聊天记录或脑海中，没有结构化梳理。

📄 只写方案还不够

方案文档只有"做什么"，没有"做出来什么样"，无法验证可行性，也无法让别人理解和复用。

🔧 缺少可复现 Demo

没有真实 Demo，无法验证方案是否可行，也无法让别人根据记录复现同类 Demo。

🔄 重复造轮子

同样的思考在不同团队、不同时间被重复，缺少共享的场景和已验证的方案。

Three Demo-Ready Blueprints

当前已收录的 3 个 demo-ready Blueprint。每个都包含真实可运行的 Demo。

Demo Pack 标准

每个完整 Blueprint 必须包含标准化的 Demo Pack：

目录内容说明
inputs/原始想法、背景材料、会议转写或需求文本保存所有输入材料
prompts/关键 Prompt驱动 Agent 的核心提示词
outputs/Brief、任务计划、生成说明Agent 生成的中间产物
app/可运行的最小 Demo真实做出来的原型
validation/验收清单判断 Demo 是否成功
README.md复现说明让别人可以照着复现

规则：没有 Demo Pack 的方案只能标记为 draft；有完整 Demo Pack 且可打开 Demo 页面，才能标记为 demo-ready。

如何使用

  1. 选择一个 Blueprint — 浏览下方 3 个方案，选择最感兴趣的
  2. 先看 Demo，再看构建过程 — 直接打开 Demo 页面体验效果，然后阅读方案页的"构建过程"章节
  3. 复制 Demo Pack，用自己的项目材料替换 inputs — 下载整个 demos/<slug>/ 目录，用你自己的输入材料替换 inputs/，重新运行 Prompt
