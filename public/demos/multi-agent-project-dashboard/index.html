AgentOps Control Tower Demo

🚀 Multi-Agent Project Dashboard 统一项目管理面板 demo-ready

统一查看多个 agent 项目的进展、风险、commit 和下一步动作。把分散在 GitHub、Telegram、报告和机器日志中的状态，整合到一个可观察的面板中。

⚠️ Demo 限制：本演示使用静态模拟数据，不会自动更新。所有项目和状态均为示例，用于展示面板结构。

📊 数据来源

  • 项目状态：模拟 6 个 agent 项目的状态报告（见 inputs/sample-agent-status-reports.md）
  • Commit 数据：模拟 git commit 列表（见 inputs/sample-git-commits.md）
  • 风险记录：通用风险 + 项目特定风险（见 inputs/sample-project-risk-notes.md）
  • 所有数据为模拟数据，请勿误认为真实项目状态

🔄 如何复现

  1. 准备多个项目阶段报告（实际使用时替换为真实数据）
  2. 提取项目名、agent、机器、阶段、状态、commit、风险、下一步
  3. 设计统一 JSON 数据结构（参考 outputs/data-model.md）
  4. 使用 prompts/dashboard-build-prompt.md 生成 Dashboard Brief
  5. 生成 data.json
  6. 替换本 Demo 的 data.json
  7. 本地打开 index.html 或发布到 GitHub Pages
  8. 根据 validation/acceptance-checklist.md 验收

⚠️ 本 Demo 的限制

  • 静态数据：不会自动更新，需要手动替换 data.json
  • 无 API 集成：不会连接 GitHub API 拉取真实 commit
  • 无历史快照：不保存历史状态，无法看趋势
  • 无通知机制：blocked 不会主动提醒
  • 无调度建议：不会主动建议下一步该做什么

📦 后续扩展

  • 接入真实 GitHub API
  • 自动解析 reports 目录
  • 接入 Telegram bot 状态消息
  • 增加历史状态快照
  • 增加项目趋势图
  • 增加 agent 调度建议
  • 与项目记忆型会议助手联动
