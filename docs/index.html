项目解决的问题

💡 想法停留在脑子里

每个人都有"如果这样做会怎么样"的想法，但大多数停留在聊天记录或脑海中，没有结构化梳理。

📄 缺少上下文

想法缺少完整的背景、问题定义和实施路径，导致其他人无法理解和复用。

🔧 没有可执行方案

大多数想法文档只有"做什么"，没有"怎么做"和"验证什么"，导致无法落地。

🔄 重复造轮子

同样的思考在不同团队、不同时间被重复，缺少共享的场景和已验证的方案。

方案库 Blueprints

新增一个方案

有一个想法？按以下步骤把它变成 Blueprint：

  1. 复制 docs/blueprint-template.md 作为起点
  2. 填写所有必填章节，保留原始想法
  3. 在 data/blueprints.json 中补充元数据
  4. 生成 HTML 页面放入 public/blueprints/ 和 docs/blueprints/
  5. 本地预览，检查链接和样式
  6. 提交并 push

什么是 Blueprint

一个 Blueprint 包含完整的可执行要素：原始想法、背景与问题、方案概述、技术架构、实施步骤、验收标准、Prompt 模板和后续扩展方向。

要素说明
原始想法记录这个想法最初是如何产生的，保留原始文本
背景与问题这个想法要解决什么问题？当前的真实痛点是什么？
方案概述核心思路，用一两句话讲清楚
技术架构最小技术栈，描述组件和它们的关系
实施步骤具体到每一步可以执行的操作
验收标准怎么判断这个 Demo 做成功了？
Prompt 模板可以直接复制给 AI Agent 使用的提示词
后续扩展验证成功后可以往哪些方向继续
