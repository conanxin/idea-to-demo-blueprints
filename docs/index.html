项目定位

这不是灵感收藏夹，而是可复现 Demo 蓝图库。

每个 Blueprint 不只记录一个想法和方案，还包含一个真实做出来的最小 Demo、完整构建过程、关键 Prompt、文件结构、验收标准和复现步骤，让其他人可以根据记录复现同类 Demo。

💡 想法停留在脑子里

每个人都有"如果这样做会怎么样"的想法，但大多数停留在聊天记录或脑海中，没有结构化梳理。

📄 只写方案还不够

方案文档只有"做什么"，没有"怎么做"和"做出来什么样"，导致无法落地和验证。

🔧 缺少可复现 Demo

没有真实 Demo，就无法验证方案是否可行，也无法让别人理解和复用。

🔄 重复造轮子

同样的思考在不同团队、不同时间被重复，缺少共享的场景和已验证的方案。

Demo Pack 标准

每个完整 Blueprint 必须包含 Demo Pack：

目录内容说明
inputs/原始想法、背景材料、会议转写保存所有输入材料
prompts/关键 Prompt驱动 Agent 的核心提示词
outputs/Brief、任务计划、生成说明Agent 生成的中间产物
app/可运行的最小 Demo真实做出来的原型
validation/验收清单判断 Demo 是否成功
README.md复现说明让别人可以照着复现

规则：没有 Demo Pack 的方案只能标记为 draft；有完整 Demo Pack 且可打开 Demo 页面，才能标记为 demo-ready。

方案库 Blueprints

新增一个 Blueprint

有一个想法？按以下步骤把它变成可复现的 Blueprint：

  1. 复制 docs/blueprint-template.md 作为起点
  2. 填写所有必填章节，保留原始想法，构建最小 Demo
  3. 创建 Demo Pack（inputs/ prompts/ outputs/ app/ validation/）
  4. 在 data/blueprints.json 中补充元数据（含 demo_url）
  5. 生成 HTML 页面放入 public/blueprints/ 和 docs/blueprints/
  6. 同步 Demo app 到 docs/demos/
  7. 本地预览，检查链接和样式
  8. 提交并 push
