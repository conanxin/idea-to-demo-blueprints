## 请输出以下内容

### 1. 客户目标摘要
用 3-5 句话概括客户的核心目标。区分明确目标、隐含目标、约束目标。

### 2. 客户痛点
列出客户描述的所有痛点。格式：痛点描述、影响、当前 workaround、期望解决方式。

### 3. 用户故事
基于客户描述，生成用户故事。格式：作为 [角色] 我想要 [功能] 以便 [价值]。验收标准、优先级。

### 4. 关键工作流
描述客户描述的核心业务流程。格式：步骤、决策点、异常流。

### 5. 待确认问题
列出会议中提出但尚未明确的问题。格式：问题描述、为什么重要、建议确认方式、优先级。

### 6. Demo 范围
明确这个 Demo 要展示什么、不展示什么。格式：包含的功能、不包含的功能、模拟的数据、假定的权限。

### 7. 非目标
明确说明这个 Demo 不做什么。格式：不是正式产品、不做真实数据集成、不做后端逻辑、不承诺交付时间。

### 8. 页面结构
设计 Demo 页面的结构。格式：页面名称、功能、关键元素、导航流、核心交互。

### 9. 数据模型
描述 Demo 中需要的数据结构。格式：实体、字段、示例数据。

### 10. Agent 构建任务
将以上分析转化为 Agent 可执行的任务清单。格式：任务、文件/组件、预期输出、依赖关系、验收标准。

### 11. 验收标准
怎么判断这个 Demo 构建成功了？标准清单。

### 12. 演示讲解稿
为产品经理/售前准备一份讲解稿。格式：开场、场景对应、核心演示、待确认、下一步。

### 13. 下一轮客户沟通问题
基于 Demo 和当前分析，生成下一轮客户沟通的问题清单。格式：确认类、优先级、技术、业务。
