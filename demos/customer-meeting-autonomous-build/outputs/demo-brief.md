# Demo Brief: 客户跟进看板

> 基于客户会议需求生成的结构化 Demo Brief
> 客户: 张总（销售总监）
> 会议日期: 2026-06-12

---

## 客户目标摘要

张总的核心目标是让销售主管能一目了然地看到每个客户的跟进状态，特别是识别风险客户和即将到期跟进的客户。隐含目标是提升团队销售效率和客户转化率。约束目标是团队只有12人、客户约200个、预算有限、不能引入复杂CRM。

---

## 客户痛点

| 痛点 | 影响 | 当前 workaround | 期望解决方式 |
|------|------|--------------|-------------|
| 不知道客户跟进到哪一步 | 销售效率低，客户流失 | 周会逐个汇报 | 一个页面看所有客户状态 |
| 主管不清楚团队整体情况 | 决策慢，无法及时介入 | 事后询问销售 | 顶部统计卡片 |
| 客户超过时间未回复无提醒 | 客户被遗忘，转化率下降 | 靠销售记忆 | 到期自动标黄色/红色 |
| CRM太复杂，团队不爱用 | 数据不更新，系统成摆设 | 部分销售用Excel | 简单页面，手动维护 |

## 用户故事

### US-001: 销售查看客户列表
- 作为 销售
- 我想要 在一个页面看到所有我负责的客户
- 以便 快速决定今天跟进谁
- 验收标准: 表格展示客户名称、阶段、风险、下次跟进时间
- 优先级: 高

### US-002: 销售主管查看团队状态
- 作为 销售主管
- 我想要 看到团队所有客户的风险分布和跟进情况
- 以便 及时发现风险并介入
- 验收标准: 顶部统计卡片 + 风险颜色标识
- 优先级: 高

### US-003: 销售更新跟进状态
- 作为 销售
- 我想要 更新客户的阶段和下次跟进时间
- 以便 团队知道最新状态
- 验收标准: 可编辑表格，更新后自动计算风险等级
- 优先级: 中（第一版可只做展示）

### US-004: 销售记录客户备注
- 作为 销售
- 我想要 记录客户的关键信息（预算、决策人、竞争）
- 以便 后续跟进有上下文
- 验收标准: 备注列可查看
- 优先级: 中

---

## 关键工作流

```
销售发现新客户 → 录入客户信息 → 设置阶段和下次跟进时间
→ 到期跟进 → 更新状态 → 成交或放弃
```

### 决策点
- 如果客户 1 周未回复 → 标黄色（有风险）
- 如果客户 1 个月未回复 → 标红色（危险）
- 如果客户成交 → 移到"成交"阶段

### 异常流
- 客户要求暂停 → 保持当前阶段，下次跟进时间清空，风险标黄色
- 客户拒绝 → 移到"放弃"阶段

## Demo 范围

### 包含
- 客户列表表格（6列：名称、阶段、风险、下次跟进时间、负责人、备注）
- 顶部统计卡片（总客户数、有风险、本周需跟进）
- 风险颜色标识（绿/黄/红）
- 模拟数据（8-10个客户）
- 简单交互（鼠标悬停显示详情）

### 不包含
- 真实数据集成
- 用户认证
- 后端逻辑
- 搜索/筛选
- 导出功能
- 移动端适配

### 模拟数据
- 8个客户，覆盖不同阶段和风险等级
- 使用真实感数据（公司名称、联系人）

## Agent 构建任务

1. 创建 HTML 结构（统计卡片区 + 表格区）
2. 创建 CSS 样式（清爽表格、风险颜色、统计卡片）
3. 创建 JavaScript 数据模型（8条客户数据）
4. 实现表格渲染（按风险排序）
5. 实现统计卡片（总客户、有风险、本周需跟进）
6. 实现风险颜色自动计算（基于下次跟进时间）
7. 添加鼠标悬停详情效果
8. 添加"原型验证"标注
9. 测试页面可打开、可交互
10. 验证数据对应客户会议中的真实表达
