# 背景与参考（Background）

> 解释为什么这个想法值得做。看完这一节，应该能讲清楚三件事：
> 1. 当前场景下做实体产品 Demo 有什么痛点
> 2. 受控定制（controlled customization）模式为什么合适
> 3. 为什么 ReadingCore-01 这种"固定内核 + 可变外壳"架构是更优解

---

## 当前场景

独立开发者和小品牌文创团队想做实体产品时，最常见的死法：

1. **每个 SKU 都从零设计** —— 成本爆炸、迭代极慢
2. **外包打样一次性消耗** —— 几百块一个样品，错了就报废
3. **结构错了重来** —— LED 灯带贴塑料外壳、电源藏不进底座、散热不够外壳发黄
4. **没有任何"可定制"的中间形态** —— 要么全定制，要么完全标准款

设计-制造-销售的链路里，**没有任何一个"想法 → 可演示原型"的快速通道**。

## 现有参考：受控定制（Controlled Customization）

行业里最经典的受控定制案例：

### Gantri

- 3D 打印灯厂
- 用一个稳定的基础结构 + 可换灯罩/颜色/灯臂
- 客户在官网选款式、价格、配色，工厂接到订单再打印
- 优点：**核心工程一次做好，只换表面**
- 缺点：整体偏氛围灯，亮度、做工偏向装饰而非阅读

### 其他类似案例

- **Bourbon Barrel Lamp / 木桶灯** — 内芯是固定 E26 灯座 + 通用 LED 灯泡，外壳是回收木桶
- **Local Motors（已停业）** — 汽车用"动力总成固定 + 车身可换"
- **Framework Laptop** — 主板固定，端口、键盘、外壳可换
- **Aulum / Gantri 衍生** — 玻璃灯罩可换

它们的共同点是：**把"不变的部分"做成强 IP，把"变的部分"做成可定制项**。

## 受控定制的代价：结构必须正确

但受控定制有个非常具体的隐藏坑：**如果"不变的内核"结构错了，整个产品线全错**。

LED 灯具有一个最常见的结构错误：

```
错误结构：
[塑料外壳] ← 直接贴 LED 灯带 → [驱动电源]

问题：
- LED 工作温度 60–80°C，长期烘烤塑料外壳
- 半年后外壳发黄、变形、有异味
- 光线是裸灯珠直发光，没有扩散，刺眼
```

正确结构（ReadingCore-01 采用）：

```
正确结构：
[24V 线性 LED] → [铝型材/铝槽散热] → [乳白扩散罩] → [可定制外壳]
                                          ↑
                          外壳只承担造型，不承担散热和电气
```

这正是用户原始想法中明确指出的"不要把灯带直接贴塑料外壳"。

## 为什么这个想法重要

1. **架构正确就是产品哲学正确** —— 把 ReadingCore-01 当成"硬件 IP"，Shell 当成"风格语言"，结构清楚
2. **Demo 可以在网页里完整表达** —— 即使没有真实 3D 打印，单页 HTML 已经能讲清楚"内核是什么、外壳怎么变、装配顺序是什么"
3. **复现门槛低** —— 任何人都可以用一份 HJT-2835 LED 灯带 + 一段铝槽 + 一片扩散片复刻
4. **AI Workflow 是天然场景** —— "北京风格阅读台灯" 这种短文本，输入后 AI 必须能自动拆成"灯芯 + 外壳"两层结构

## 这跟"配置器"的区别

常见配置器（如 Nike By You、自行车配置器）：

- 是在已知 SKU 上的**配色选择**
- 没有架构拆分
- 不能让 AI 从一句想法里推导出结构

本 Demo 的关键不同：

- AI 必须先做"产品架构推理"（决定 ReadingCore-01 适不适合）
- 然后才是"配置"（选外壳）
- 最后是"制造计划"（输出可执行的 BOM + 装配步骤）
