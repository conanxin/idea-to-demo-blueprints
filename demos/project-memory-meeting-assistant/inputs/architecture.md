# TeamFlow Dashboard — 架构文档

```
┌─────────────────────────────────────────┐
│              前端层（React SPA）          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │

│  └──────────┘ └──────────┘ └──────────┘ │
└──────────────────┬──────────────────────┘
                   │ REST API
┌──────────────────┴──────────────────────┐
│              后端层（Node.js）            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 任务服务  │ │ 风险服务  │ │ 会议服务  │ │
│  └──────────┘ └──────────┘ └──────────┘ │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────┴──────────────────────┐
│              数据层（PostgreSQL）         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
```

## 关键设计决策

- **前端**：单页应用，使用 React Router 管理路由
- **状态管理**：React Context + useReducer（足够轻量，不需要 Redux）
- **API 设计**：RESTful，JSON 格式
- **认证**：JWT Token，存储在 localStorage
- **数据库**：单一 PostgreSQL 实例，使用 Sequelize ORM

## 技术约束

- 不使用 GraphQL（团队不熟悉，学习成本高）
- 不使用微服务（团队规模小，维护成本太高）
- 前端不使用服务端渲染（SSR 对 SEO 不重要，这是内部工具）
